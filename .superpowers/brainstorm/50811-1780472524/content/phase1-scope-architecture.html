Phase 1 Scope: Android and iOS Parity

First release target: normal recording, loop segments, video playback, and persistent settings on both platforms. Motion detection is deferred.

Recommended Architecture

RecordingViewModel

Business rules live here, not inside platform Composables.

Platform Engines
Android CameraX Recorder
iOS AVFoundation Recorder
Platform Storage
Platform Player Surface

Platform code implements capabilities through expect/actual interfaces.

A

Single Recorder Controller

One shared controller coordinates normal and loop recording. Platform engines expose startSegment/stopSegment. Recommended for parity and low coupling.

B

Separate Mode Controllers

Normal recording and loop recording each get a controller. Easier at first, but shared state and transitions can drift.

C

Platform-Owned Recording

Android and iOS manage recording independently. Fastest to patch current code, weakest for long-term consistency.

Recommendation

Use A. Single Recorder Controller. It keeps the UI identical, makes loop recording a policy in shared code, and keeps CameraX / AVFoundation implementation details behind small platform interfaces.
